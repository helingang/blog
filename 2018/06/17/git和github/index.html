什么是github

一个免费的远程代码仓库

分布式和集中式

  1. 集中式
    1. 代码集中在一个中央服务器上,所有代码提交到一个服务器中进行管理,代表(SVN)
    2. 缺点:中央服务器故障
  2. 分布式
    1. 每个人可在本地创建代码仓库,可以与远程代码仓库同步代码,也可以互相同步代码
    2. 优点:
      1. 每一次提取操作,实际都是对代码仓库的完整备份,git中每一个克隆库都是平等的,可以从任何一个版本库的克隆来创建属于你自己的版本库
      2. 可以脱离git管理代码
      3. 提交在本地完成,提交旧版本会创建分支
      4. git提交不会被打断,不能自动同步的冲突提示手动完成
