JavaScript中正则的使用

发表于 2018-06-16 | 分类于 JavaScript

JavaScript正则的使用

  1. 可以用正则的方法
    1. reg.exec(str);返回查找的字符串或null,但是只能匹配一次不能全局匹配;返回一个数组,包含匹配到的字符串和分组规则匹配的值
    2. reg.test(str);返回true或false
    3. str.match(reg);返回查找的字符串数组或null,匹配一个;加g时才返回标准数组;不加g时返回类数组包含结果和子集
    4. str.replace(reg , str1);str1可以用函数代替function(a,b,c,d){}a: 匹配内容 b: 匹配的子集 c:匹配内容出现的序号 d: 整体str
    5. str.search(reg);返回查找匹配字符串首次出现的下标或-1
    6. str.split(reg);
  2. 创建正则表达式
    1. 直接量
      var reg = /a/igm;
    2. 构造函数
      var reg = new RegExp(/\w/,'igm');
      var reg = new RegExp('\\w','igm');需要转义
  3. 修饰符
    1. i 忽略大小写
    2. g 全局匹配(查找所有匹配,而非在找到第一个后停止),加了g后返回标准数组
    3. m 执行多行匹配
  4. 特殊字符
    1. |表示或
      1. var reg = new RegExp(/abc|def/,'igm');
    2. 字符集[]匹配括号中任意一个字符
      1. var reg = new RegExp(/[java]/,'igm');
      2. 从str中依次遍历j,a,v,a(匹配j或a或v或a)
      3. []中任何符号都会被认作匹配字符
      4. [a-zA-Z0-9]匹配一个数字或字母
      5. [^0-9]匹配一个非0-9的word
      6. [\u4e00-\u9fa5]匹配一个中文
    3. ^与$
      1. /^java\w*2017$/;//匹配以java开头,2017结尾的字符
      2. /^a/;//以a开头
      3. /a$/;//以a结尾
      4. 注^在[]才表示非
    4. 元字符(转义字符)
      1. \ 转义符(将一个具有特殊意义的字符变得没有意义)
        1. \n字符串换行符 \t制表符 \r回车符
      2. \w 匹配一个[0-9A-Za-z_];word
      3. \W 匹配一个[^0-9A-Za-z_]
      4. \d 匹配一个[0-9];digital
      5. \D 匹配一个[^0-9]
      6. \s 匹配一个空格
      7. \S 匹配一个非空格
      8. \b 匹配一个 单词边界(除了\w之外的都可充当单词边界)
      9. \B匹配不在单词边界上的字符
      10. .匹配所有字符(除了字符串换行符\n,注意不是.)

  5. 量词(针对前面一个符号(或子集))

    1. n? 匹配0个或1个n
    2. n* 匹配0个,1个或多个n
    3. n+ 匹配1个或多个n
    4. n{x} 匹配x个n
    5. n{x,y} 匹配x~y个n
    6. n{x,} 匹配至少x个n

  6. 贪婪和惰性(针对量词)
    量词默认贪婪(尽可能多的匹配)
    如要添加惰性,则?后面不能添加东西

尽可能多的匹配 尽可能少的匹配
+(匹配一个或多个) +?(选择匹配1个)
*(匹配0个一个或多个) *?(选择匹配0个
{1,20} {1,20}?(选择匹配1个)

  1. 分组

    1. 捕获型分组
      1. \1
        1. (n)m\1 \1引用第一个分组( (n) )匹配的结果,第x个左括号就是第x个匹配规则
      2. RegExp.$1
        1. 获取第一个分组匹配的结果
    2. 非捕获型分组(一种断言)
      1. 正向前瞻
        1. var reg = /javascript(?=2007)/;匹配后面是2007的javascript(只拿到javascript);(?:2007)
      2. 负向前瞻
        1. var reg = /javascript(?!2007)/;匹配后面不是2007的javascript
      3. 正向后顾
        1. ?<=2007
      4. 负向后顾
        1. ?<!2007
